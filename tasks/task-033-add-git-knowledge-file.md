# Task-033: Add Git and .gitignore Policy to Knowledge Base

**Location:** `knowledge/`

**Description:**
1. Create a dedicated knowledge base file `knowledge/git.md` explaining the project's Git and `.gitignore` policies.
2. Outline the principles of operation for `.gitignore`:
   - Ignore temporary directories (such as `tmp/`).
   - Ignore directories containing downloaded package/library code (e.g. `node_modules/`, `vendor/`), as they are dynamically generated by package managers.
   - Do NOT ignore configuration/manifest files describing these dependencies (e.g. `package.json`, `composer.json`, `requirements.txt`, lockfiles), as they represent the source of truth for dependencies.
3. Update `file-templates/MANIFEST.md` or any other structures if needed to reflect the new file.

**Justification:**
Establishes a clear, cohesive reference for AI agents and developers regarding how the project handles version control and dependency exclusion in `.gitignore`.

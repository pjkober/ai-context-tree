# Task-085: Add Semantic Versioning to the Standard

## Source
Report section 7.1 (Missing element).

## Location
- `VERSION` (new file at root)
- `file-templates/AGENTS.md` -- add a version field
- `file-templates/MANIFEST.md` -- add a version reference
- `README.md` -- add version badge and section

## Description

`ai-context-tree` is a published standard. Standards evolve. Currently there is no version identifier anywhere in the repository. Adopters have no way to:
- Know which version of the standard they are using.
- Determine if a breaking change in the template was introduced after they adopted it.
- Communicate "we use ai-context-tree v0.5" in their project documentation.

## What to Do

### 1. Create `VERSION` at the project root

```
0.6.0
```

Single-line file with the semantic version. Follow [Semantic Versioning 2.0.0](https://semver.org):
- MAJOR: Breaking changes to directory structure or AGENTS.md format.
- MINOR: New templates, new knowledge files, backward-compatible improvements.
- PATCH: Bug fixes, typo corrections, documentation-only changes.

### 2. Update `README.md`

Add a version badge next to the existing badges:
```markdown
[![Version](https://img.shields.io/badge/version-0.6.0-blue.svg)](VERSION)
```

Add a brief "Versioning" section near the bottom:
```markdown
## Versioning

This standard follows [Semantic Versioning](https://semver.org). The current version is in the [VERSION](VERSION) file.

- MAJOR versions introduce breaking changes to directory structure or AGENTS.md format.
- MINOR versions add new templates or guides (backward-compatible).
- PATCH versions fix typos or documentation errors.
```

### 3. Add version to `file-templates/AGENTS.md`

Add a comment line at the top:
```markdown
# AGENTS.md
# Generated by ai-context-tree standard v__VERSION__
```

The bootstrap script should substitute `__VERSION__` with the current version from the `VERSION` file.

### 4. Add version substitution to bootstrap scripts

In `create_minimal_structure.sh` and `create_minimal_structure.ps1`, when generating `AGENTS.md`, read the `VERSION` file and substitute `__VERSION__` in the output.

### 5. Add `VERSION` to project `MANIFEST.md`

Add `├── [VERSION](VERSION)` to the manifest listing.

## Verification

- `VERSION` file exists at project root with a valid semantic version string.
- `README.md` displays the version badge.
- `README.md` contains a "Versioning" section.
- The bootstrap script substitutes the version into generated `AGENTS.md` files.
- `cat VERSION` returns a valid semver string (e.g., `0.6.0`).

# Coding Standards

Follow these coding conventions for all modifications to this Node.js repository.

## General Guidelines

- **File Indentation:** Use tabs (not spaces) for indentation to follow project format.
- **Quotes:** Use double quotes for strings in TypeScript files.
- **Function Limit:** Keep functions under 40 lines of code.
- **File Limit:** Keep files under 300 lines. If a file exceeds this limit, split it following the single-responsibility principle.
- **Refactoring:** Boy Scout Rule — Fix minor code smells in files you are editing, but avoid refactoring symbols outside the task scope.
- **Linter:** Run `npm run lint --fix` before committing changes.

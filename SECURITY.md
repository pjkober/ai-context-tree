# Security Policy

## Supported Versions

`ai-context-tree` is a documentation and tooling standard, not executable application code. The bootstrap scripts (`create_minimal_structure.sh`, `create_minimal_structure.ps1`) are the only executable components.

| Version | Supported |
|---|---|
| Latest (main branch) | Yes |
| Older tagged versions | No |

We recommend always using the latest version from the `main` branch.

---

## Reporting a Vulnerability

If you discover a security vulnerability in the bootstrap scripts or any other component of this project, please do NOT open a public GitHub issue.

Instead, report it privately via one of these methods:
1. **GitHub Private Vulnerability Reporting:** Use the "Report a vulnerability" button on the [Security tab](../../security/advisories/new) of this repository.
2. **Email:** Contact the project maintainer directly (see the GitHub profile for contact information).

Please include:
- A clear description of the vulnerability.
- The file(s) and line number(s) affected.
- Reproduction steps (if applicable).
- Potential impact assessment.

---

## Security Best Practices for Bootstrap Script Users

When running the bootstrap scripts in your project:

1. **Never commit secrets:** The `ai/rules/security.md` template explicitly prohibits committing API keys, passwords, or tokens. Use environment variables and a secrets manager instead.

2. **Review generated files:** Before committing the generated project structure to a repository, review the generated `ai/rules/security.md` and confirm the placeholder values match your actual security policy.

3. **Autonomy Mode users:** If you configure your AI agent in full autonomy mode, ensure your `.gitignore` correctly excludes all credential files before granting write access to the repository.

---

## Scope

This security policy covers:
- The Bash bootstrap script (`create_minimal_structure.sh`).
- The PowerShell bootstrap script (`create_minimal_structure.ps1`).
- Template files distributed with this repository.

It does NOT cover security of the target projects generated by the scripts. That is the responsibility of the adopting team.

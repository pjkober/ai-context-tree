# Living Specifications

**Living Specifications** are documentation files (typically in `specs/`) containing the requirements and acceptance criteria for a feature. Unlike static documentation that becomes outdated immediately after a feature is built, a living specification is continuously updated alongside the implementation.

In an AI-First workspace, these files serve as the Single Source of Truth (SSOT). They are written in structured checklist formats that AI agents can parse. In the [Spec-Driven Development](Spec-Driven%20Development.md) lifecycle, the AI agent uses these specifications to generate unit/integration tests and verify that all requirements have been met.

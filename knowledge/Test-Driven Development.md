# Test-Driven Development (TDD)

**Test-Driven Development (TDD)** is a software development process where requirements are turned into very specific test cases, and then the software is improved so that the tests pass. The cycle consists of: writing a failing test, writing code to make the test pass, and refactoring the code.

In the `ai-context-tree` standard, the developer configures the AI's testing rules:
1. **Test-First (TDD):** The AI agent is instructed to write test files and definitions first, before implementing the feature logic.
2. **Post-Implementation (Write-After):** The AI agent implements the feature code first, and then writes the corresponding tests to verify the implementation.
